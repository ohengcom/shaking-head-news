# 生产环境测试指南

本文档提供 Vercel 部署后的完整测试流程。

---

## 📋 测试前准备

### 1. 获取部署信息

- **生产 URL**: `https://your-domain.vercel.app`
- **Vercel 项目**: 登录 [Vercel Dashboard](https://vercel.com/dashboard)
- **环境变量**: 确认所有环境变量已配置

### 2. 准备测试工具

- **浏览器**: Chrome, Firefox, Safari, Edge
- **移动设备**: 或使用浏览器开发者工具模拟
- **测试账号**: Google 账号用于登录测试

---

## 🧪 测试流程

### 阶段 1: 自动化基础测试

运行自动化测试脚本：

```powershell
# 替换为你的实际 URL
.\scripts\test-production.ps1 -Url "https://your-site.vercel.app"
```

这将检查：

- ✅ 网站可访问性
- ✅ 关键页面状态
- ✅ 安全响应头
- ✅ 资源加载

---

### 阶段 2: 功能测试

#### 2.1 首页和新闻功能

**测试步骤**:

1. 访问首页
2. 检查新闻是否显示
3. 点击"刷新"按钮
4. 点击新闻链接，验证是否打开正确页面

**预期结果**:

- ✅ 新闻列表正常显示
- ✅ 刷新功能正常工作
- ✅ 新闻链接可以打开
- ✅ 加载状态显示正确

**常见问题**:

- ❌ 新闻不显示 → 检查 NEWS_API_BASE_URL 环境变量
- ❌ 刷新失败 → 检查 ISR 缓存配置
- ❌ 加载很慢 → 检查 Vercel 区域设置

---

#### 2.2 页面旋转功能

**测试步骤**:

1. 观察页面是否有旋转动画
2. 点击"暂停"按钮
3. 点击"继续"按钮
4. 切换"固定模式"和"连续模式"
5. 调整旋转间隔滑块

**预期结果**:

- ✅ 页面有平滑的旋转动画
- ✅ 暂停/继续功能正常
- ✅ 模式切换生效
- ✅ 间隔调整生效

**常见问题**:

- ❌ 没有动画 → 检查浏览器是否启用了"减弱动态效果"
- ❌ 动画卡顿 → 检查浏览器性能
- ❌ 控制无效 → 检查浏览器控制台错误

---

#### 2.3 用户认证

**测试步骤**:

1. 点击"登录"按钮
2. 选择 Google 账号登录
3. 授权应用访问
4. 验证是否跳转回首页
5. 检查是否显示用户头像/名称
6. 点击"登出"

**预期结果**:

- ✅ Google OAuth 登录流程顺畅
- ✅ 登录后显示用户信息
- ✅ 登出功能正常

**常见问题**:

- ❌ 登录失败 → 检查 Google OAuth 配置
- ❌ 回调错误 → 检查 OAuth 回调 URL 和部署域名配置
- ❌ 会话丢失 → 检查 AUTH_SECRET 配置

**检查清单**:

```
Google Cloud Console 配置:
[ ] 已添加生产环境回调 URL
[ ] OAuth 同意屏幕已配置
[ ] 客户端 ID 和密钥正确

Vercel 环境变量:
[ ] AUTH_GOOGLE_ID
[ ] AUTH_GOOGLE_SECRET
[ ] AUTH_SECRET
[ ] NEXT_PUBLIC_APP_URL
[ ] 如需 Pro 白名单，已配置 PRO_USER_IDS / PRO_USER_EMAILS
```

---

#### 2.4 设置管理

**测试步骤**:

1. 打开设置面板
2. 切换主题（浅色/深色/系统）
3. 调整字体大小
4. 切换语言（中文/英文）
5. 修改其他设置
6. 刷新页面，检查设置是否保存

**预期结果**:

- ✅ 所有设置立即生效
- ✅ 刷新后设置保持
- ✅ 登录用户设置云端同步

**测试云端同步**:

1. 登录账号
2. 修改设置
3. 在另一个浏览器/设备登录同一账号
4. 验证设置是否同步

**常见问题**:

- ❌ 设置不保存 → 检查 LocalStorage 或 Upstash Redis
- ❌ 云端同步失败 → 检查 UPSTASH_REDIS_REST_URL 和 TOKEN
- ❌ 主题切换无效 → 检查 next-themes 配置

---

#### 2.5 RSS 源管理

**测试步骤**:

1. 进入 RSS 管理页面
2. 点击"添加 RSS 源"
3. 填写 RSS 信息：
   - 名称: "测试 RSS"
   - URL: "https://example.com/feed.xml"
   - 语言: 中文
4. 保存并验证是否显示在列表中
5. 启用/禁用 RSS 源
6. 删除 RSS 源
7. 测试 OPML 导出

**预期结果**:

- ✅ 可以添加自定义 RSS 源
- ✅ RSS 源列表正确显示
- ✅ 启用/禁用功能正常
- ✅ 删除功能正常
- ✅ OPML 导出成功

**常见问题**:

- ❌ 添加失败 → 检查 RSS URL 格式
- ❌ 数据不保存 → 检查 Upstash Redis 连接
- ❌ OPML 导出失败 → 检查浏览器控制台错误

---

#### 2.6 统计功能

**测试步骤**:

1. 进入统计页面
2. 检查是否显示统计数据
3. 验证图表是否正确渲染
4. 测试健康提醒通知（需要授权）

**预期结果**:

- ✅ 统计数据正确显示
- ✅ 图表渲染正常
- ✅ 通知功能正常（如果授权）

**常见问题**:

- ❌ 数据不显示 → 需要先使用一段时间积累数据
- ❌ 图表不渲染 → 检查 Recharts 库加载
- ❌ 通知不工作 → 检查浏览器通知权限

---

### 阶段 3: 性能测试

#### 3.1 Lighthouse 测试

1. 访问 [PageSpeed Insights](https://pagespeed.web.dev/)
2. 输入你的网站 URL
3. 运行测试

**目标指标**:

- **Performance**: > 90
- **Accessibility**: > 95
- **Best Practices**: > 90
- **SEO**: > 90

**Core Web Vitals**:

- **LCP (Largest Contentful Paint)**: < 2.5s
- **FID (First Input Delay)**: < 100ms
- **CLS (Cumulative Layout Shift)**: < 0.1

#### 3.2 Vercel Analytics

1. 登录 Vercel Dashboard
2. 进入你的项目
3. 查看 Analytics 标签

**检查项**:

- ✅ Real Experience Score
- ✅ Web Vitals 数据
- ✅ 访问量统计
- ✅ 地理分布

#### 3.3 加载速度测试

使用浏览器开发者工具：

1. 打开 Chrome DevTools (F12)
2. 切换到 Network 标签
3. 刷新页面
4. 检查：
   - **DOMContentLoaded**: < 1.5s
   - **Load**: < 3s
   - **总请求数**: < 50
   - **总大小**: < 2MB

## 🔧 常见问题排查

### 问题 1: 网站无法访问

**可能原因**:

- Vercel 部署失败
- DNS 配置错误
- 域名未生效

**排查步骤**:

1. 检查 Vercel Dashboard 部署状态
2. 查看 Vercel 部署日志
3. 检查域名 DNS 设置
4. 尝试使用 Vercel 默认域名访问

### 问题 2: 环境变量未生效

**可能原因**:

- 环境变量未配置
- 环境变量名称错误
- 需要重新部署

**排查步骤**:

1. 检查 Vercel 项目设置 → Environment Variables
2. 确认变量名称正确（区分大小写）
3. 确认变量值正确（无多余空格）
4. 重新部署项目

### 问题 3: Google 登录失败

**可能原因**:

- OAuth 回调 URL 未配置
- 客户端 ID/密钥错误
- OAuth 回调 URL 或部署域名配置错误

**排查步骤**:

1. 检查 Google Cloud Console OAuth 配置
2. 确认回调 URL: `https://your-domain.com/api/auth/callback/google`
3. 检查 Vercel 环境变量
4. 查看浏览器控制台错误信息

### 问题 4: 数据不保存

**可能原因**:

- Upstash Redis 未配置
- Redis 连接失败
- 用户未登录（云端同步）

**排查步骤**:

1. 检查 Vercel Storage 是否已添加
2. 检查 UPSTASH_REDIS_REST_URL 和 TOKEN
3. 查看 Vercel 函数日志
4. 测试 Redis 连接

### 问题 5: 性能差

**可能原因**:

- ISR 缓存未生效
- 图片未优化
- 区域设置不当

**排查步骤**:

1. 检查 Next.js 缓存配置
2. 使用 Lighthouse 分析性能瓶颈
3. 检查 Vercel 区域设置
4. 优化图片和资源加载
